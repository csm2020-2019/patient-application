get()

get(  $id) : array|null

Get Patient By User ID

Parameters

$id

Returns

array|null —

This method takes a User ID, and uses it to grab a Patient object using a static method. This object is then used to construct an associative array of very select information that the frontend can display as read-only, or be inserted for use in forms, like the patient's address or email address. If anything goes wrong, null is returned.

Address Method

Parameters

$address2
$town
$postcode
$uid

Returns

boolean|null —

When an address is provided to the backend to be changed, after sanitisation and validation, the aspects of the address are concatenated into a master string for insertion into the database.

This is done by retrieving a Patient object using the current user's ID, and if there's no funny business, setting the address property and updating the changes accordingly. If it all goes swell, true is returned. Otherwise, a null value is returned.

validateAddress()

validateAddress(array  $components) : boolean

Validate Address

Parameters

array $components

Returns

boolean —

This small method checks that none of the inputs are null and are neither empty, or too small or too long. This is also enforced client-side, so this mainly exists to circumvent any shenanigans. False is returned on any failure of validation. True if everything is okay.

email()

email(  $email,   $uid) : boolean|null

Email Method

Parameters

$email
$uid

Returns

boolean|null —

This method is used for changing the user's email address. After sanitisation and validation, in the same way as the address method works - a Patient object is insantiated using the current user ID, the property is modified, and then the changes are saved to the database. Any funny business or failure returns null. True is returned if everything goes swimmingly.
